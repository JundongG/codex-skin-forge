# 客户主视觉制作流程

## 路线 A：客户提供图片

1. 获取尽量高清的原图并确认用途授权。
2. 根据 1600×760 横向画布扩图或重新构图。
3. 主体通常放在右侧，避开输入框、建议卡和窗口按钮；左侧保留标题安全区。
4. 按客户风格完成调色、背景、光效、纹理、边框和品牌装饰。
5. 输出预览，确认人物裁切、品牌元素、文案和签名。
6. 导出最终 PNG/JPEG，再进入客户包构建。

## 路线 B：客户只提供想法

1. 把想法拆成主体、场景、风格、颜色、材质、光线、构图和禁用元素。
2. 先制作 2–3 个低成本方向稿，不直接打包第一张生成图。
3. 客户选定方向后，统一人物/品牌细节，修正手部、文字、边缘和背景结构。
4. 按 Codex 首页布局制作 1600×760 最终构图，必要时使用局部重绘或扩图。
5. 客户确认并完成素材权利声明后，再进入客户包构建。

## 打包命令

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\create-client-package.ps1 `
  -ClientId "client-id" `
  -ClientName "客户名称" `
  -HeroImage "C:\path\customer-final.png" `
  -Tagline "客户确认的副文案" `
  -Signature "客户签名" `
  -Badge "EXCLUSIVE" `
  -AssetSource "客户提供并确认授权的照片，设计方完成扩图和排版" `
  -ConfirmAssetRights
```

打包器只接收已经确认的最终图。它不会自动把临时占位图、生成草稿或聊天截图带进客户交付包。
