# 测试矩阵

## 自动化门禁

`tests/build-and-verify.ps1` 使用临时生成的 1600×760 测试图完成以下检查，结束后删除测试包，不安装或启动 Codex：

- 所有 PowerShell 脚本语法检查。
- CDP 固定回环地址、`app://` 目标限制和装饰层点击穿透静态检查。
- 安装脚本禁止网络下载原语。
- 客户主视觉必填、最低尺寸、文件类型和 16 MiB 限制。
- 未确认素材权利、模板标记未清除时拒绝构建。
- ZIP 必需文件、主视觉哈希和全部 SHA-256 校验。
- 客户包不包含 `third_party`、Node、Codex 或占位说明。

## 真机矩阵

| 场景 | 预期结果 |
|---|---|
| Windows 11 + 当前 Store Codex | 安装、交接启动、验证和恢复通过 |
| Windows 10 + 当前 Store Codex | 同上；窗口圆角差异不影响操作 |
| Codex 未安装 | 预检停止，不写入安装目录 |
| 客户主图缺失或被替换 | SHA-256 失败，拒绝安装 |
| Node 签名无效或版本过低 | 安装停止，不启动 CDP |
| 安装时 Codex 正在运行 | 安装完成后显示交接提示，等待客户手动关闭 |
| 客户不关闭 Codex | 超时退出，不强制结束进程 |
| 随机端口被占用 | 重新选取空闲回环端口 |
| 首页 | 主视觉、客户文案、建议卡和输入框无遮挡 |
| 任务页 | 视觉弱化，不覆盖代码、终端或正文 |
| 设置弹窗 | 文本和控件可读，弹窗层级高于装饰层 |
| 代码块和 diff | 语义色可辨，不被金色主题吞没 |
| 100/125/150% 缩放 | 无横向溢出，核心控件可操作 |
| 小窗口 1024×700 | 主图裁切合理，装饰自动收敛 |
| 减少动态效果 | 动画关闭，布局保持 |
| 注入器异常退出 | Codex 本身继续运行；重启专属快捷方式可恢复 |
| 恢复原版 | 当前注入移除；关闭该窗口后调试端口结束 |
| 卸载 | 删除本产品引擎、主题、复制运行时和快捷方式，不动 Codex |

## 每个客户的视觉验收

- 主体没有被裁掉，脸部/品牌标志不落在按钮和输入框后面。
- 左侧标题安全区清晰，文案与背景对比足够。
- 客户名、签名、徽章和文案无错字。
- 图片来源、授权确认和最终文件哈希与交付记录一致。
- 验收截图不包含私密聊天内容或账号信息。
