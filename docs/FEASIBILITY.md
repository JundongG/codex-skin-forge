# 可行性结论

## 能做到什么

官方 Codex 外观设置适合基础配色、字体和代码主题；要实现营销图级别的客户主视觉、装饰框、签名和首页卡片重排，需要额外的渲染层。

Windows 上可通过官方 Codex 的 Chromium CDP 启动参数，在不修改应用文件的前提下把本地 CSS、DOM 和客户图片注入 `app://` 渲染页面。`Fei-Away/Codex-Dream-Skin` 已公开展示这一方向；本项目用独立实现补齐客户模板、无联网安装、签名运行时复制、随机回环端口、自动打包、校验和恢复。

## 适合商业交付的条件

- 每个客户单独设计主视觉，不复用未授权人物图。
- 主视觉与文案经过客户确认并记录哈希。
- 安装包不修改 Codex 文件，不声称官方合作。
- 目标 Codex 版本完成真机回归。
- 更新后选择器不兼容时，优先恢复原版并更新皮肤，而不是强行打补丁。

## 当前范围

第一阶段支持 Windows 10/11 Microsoft Store Codex。macOS 的应用启动、签名和分发边界不同，不能直接复用 Windows 安装包，需要单独开发与验收。
